# Правила работы для AI-ассистента (svoi-la)

## Контекст
Next.js 14 (App Router) + Supabase + Anthropic + Sentry. PWA для иммигрантов в LA.
Деплой — Vercel. Главный файл `src/components/SvoiApp.jsx` — рефакторится из God-component'а.
Подробное состояние: `REFACTOR_NOTES.md` в корне.

## Workflow

1. **Никогда не работай в `main`.** Создавай ветку `refactor/...` или `feature/...` от main.
2. **Один коммит = одна тема.** Не пихай рефакторинг + фикс + фичу в один коммит.
3. **Между шагами — smoke-тест:** логин → карточка места → лайк → чат → карта → форма.
4. **После каждого шага — мерж в main через PR на GitHub.** Не копи 10 коммитов в ветке.
5. **Я не пушу за пользователя** — sandbox не имеет git-доступа. Готовлю команды, пользователь выполняет в PowerShell или GitHub Desktop.

## Код

1. **ESLint `react-hooks/exhaustive-deps: "error"`** — Vercel build падает на этом. Чинить **правильно** (в deps + useCallback), не отключать.
2. **Сеттеры из `useState`** стабильны, можно добавлять в deps без страха цикла.
3. **Сеттеры из кастомных хуков** ESLint считает нестабильными — добавлять в deps вручную.
4. **Pure функции** вынеси за пределы компонента (модульный уровень) — ESLint не ругается, deps не нужны.
5. **Не оборачивай всё в `useCallback` подряд** — только то, что используется в deps других хуков или мемоизированных компонентов.

## Архитектура

1. **Не ломай существующее API** — обёртки, экспортируемые из `lib/maps.js`, `lib/views.js` и хуков, должны сохранять сигнатуру при правках.
2. **Серверные данные** идут через `useAppData` хук. UI-state — пока в `useState` SvoiApp, но Шаг 5 переедет в Zustand.
3. **Карточки выноси в `src/components/svoi/cards/*.jsx`** перед редизайном. Не правь дизайн inline в SvoiApp.
4. **SQL миграции в `sql/migrations/`** — пронумерованы, идемпотентны (`IF NOT EXISTS`, `ON CONFLICT DO NOTHING`).
5. **Не коммить service-role-ключи** и `.env.local`. Они уже в `.gitignore`.

## Sandbox-ограничения

1. **Не могу удалять файлы.** Если нужно удалить — пишу пользователю PowerShell-команду (`Remove-Item ...`).
2. **Не могу делать `git`-операции.** Готовлю команды, пользователь сам коммитит.
3. **Bash-mount стейл.** Если `Read` показывает X, а `bash cat` показывает Y — верь `Read`.
4. **Файлы с BOM/CRLF** в репо — норма, не «правь».

## Чего НЕ делать

1. **Не отключать ESLint правила** для обхода — кроме `@next/next/no-img-element` (отключено осознанно до миграции на `next/image`).
2. **Не делать "полную перезапись"** компонентов — рефакторь поэтапно. Каждый шаг — шиппабельный.
3. **Не смешивать рефакторинг с фичами** в одной ветке.
4. **Не доверять клиентским данным** в API-routes (`/api/chat` уже это игнорирует).
5. **Не использовать `localStorage`/`sessionStorage` без `try/catch`** — Safari в private mode бросает SecurityError.
6. **Не использовать `alert()`/`confirm()`** в новом коде — есть backlog задача заменить на тосты.

## Стиль ответов

1. **Коротко и по делу.** Если можно сказать в 3 строки — не пиши 30.
2. **Списки `1. 2. 3.`** только для конкретных action-items.
3. **Код-блоки с командами** (PowerShell) — для всего, что пользователь должен выполнить руками.
4. **После изменений — кратко описать что сделано + что коммитить + что протестить.**
5. **Если упёрся в неоднозначность — спрашивай ОДНИМ вопросом**, не пиши «варианты A/B/C» каждый раз.

## Что прочитать перед стартом

1. `REFACTOR_NOTES.md` — полное состояние проекта и backlog.
2. `package.json` — версии и скрипты.
3. `.eslintrc.json` — текущие правила.
4. `sql/migrations/` — последний номер миграции, чтобы новая шла следующим.
